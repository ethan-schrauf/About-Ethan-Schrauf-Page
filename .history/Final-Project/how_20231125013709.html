How To Do It

Several experts emphasize that Universal Design should be approached as a design philosophy rather than a rigid checklist or a set of predefined solutions, measurements, or products. On this page, we aim to establish a step-by-step method for achieving Universal Design on the web. To initiate this process, we will explore fundamental design principles outlined by cognitive scientists and design authorities. One key influence in this domain is Donald Norman, whose significant impact on design practices is evident in his widely acclaimed work, "The Design of Everyday Things" (1988). This page will delve into Norman's design concepts, providing a shared and adaptable language for discussing design principles in both physical and virtual contexts.

Percieved Affordance

Although Norman did not coin the term or originate the idea of "affordance," he is acknowledged for introducing the concept of "perceived affordance," which pertains to how users interpret and perceive the function of a device. This distinction is subtle but crucial in effective design, where the affordance should not only align with the intended function but also be clearly visible to the user.

In well-designed instances, especially for simple objects like a pencil, hammer, or scissors, we are often unaware of the cues a device provides, indicating sound design. If we find ourselves having to consciously think about how to operate a device, it typically means that its "perceived affordance" is not readily apparent to the user, or the device's affordance does not align with its function. This concept of affordance extends to electronic devices and computer interfaces, and throughout this chapter, we will explore how both physical objects and web pages can either accurately convey cues for operation or mislead the user.

In later editions of "The Design of Everyday Things" (DOET), Norman discusses how readers have shared numerous examples of "Norman Doors" or "Norman devices." These refer to doors or devices whose affordances convey incorrect cues for operation. For instance, a door handle suggesting pulling when pushing is required, or a handle designed for pulling with a shape that signals pushing due to its wide, flat design. Constraints

Constraints serve to restrict user actions, aiming to prevent errors and the misuse of a device. Unlike affordances, which guide users toward specific operations, constraints limit a system's choices to avoid errors, protect the device from damage, and prevent harm to the user or others. By establishing boundaries, constraints direct users toward the correct operation or usage of a system. Using examples from the automotive context, constraints include features such as the inability to remove a key from a car ignition until the engine is turned off, the incapability of shifting the transmission into reverse while the car is in forward motion, and the audible chime that sounds when attempting to exit the car with the key still in the ignition.

Visibility and Feedback

Norman emphasizes that visible implementation of physical constraints is crucial to dissuade incorrect operations, as seen in a gas station credit card device with a diagram preventing incorrect card insertion. Visibility extends to hiding inappropriate options, as observed in computer software where unavailable menu options and grayed-out non-focused windows align with the current system state. In effective design, balancing visibility and invisibility is achieved through hierarchical menu systems and organized information, avoiding overwhelming users with excessive options. The concept of "context sensitivity," akin to "progressive disclosure" in web design, involves revealing or hiding controls based on the user's context within the system. Additionally, presenting basic operations initially with the option to unveil more complex features upon user request, such as Google's "advanced search," creates a simpler yet flexible interface catering to various user levels.

Mapping

The final design principle discussed in this chapter is mapping, which involves presenting controls that reflect the operation to be performed or, through positioning or proximity, indicate the device or component associated with the control. A straightforward example of this is the placement of light switches near the lights they operate; we instinctively associate the closest switch with the corresponding light. What Are Web Standards?

What do we mean by Web Standards? Jeff Zeldman, the author of "Designing with Web Standards" (2009) and co-founder of the Web Standards Project (WaSP), defines Web Standards as the use of structural languages like HTML, XHTML, and XML, presentation languages like CSS, object models like the W3C DOM, and scripting languages like JavaScript. In simpler terms, Web Standards entail creating web pages with syntax and code elements recognized by various browsers and platforms that adhere to standards, including IE 9+, Apple's Safari 3.0+, Firefox 3+, Chrome, and Opera 9+. This involves using one of the Doctypes listed at the top of the HTML or HTM file and validating the code to ensure compliance.

Web Standards also include the separation of content from presentation, utilizing (X)HTML for structure and content, and CSS for styling, while avoiding tables for layout and employing semantically correct code. Non-standards-compliant sites, such as those using proprietary coding like Amazon, highlight the importance of adhering to standards.

Before the adoption of Web Standards, many websites used browser-specific proprietary coding techniques, resulting in code-forking and increased development costs. While browser-specific issues are largely resolved, non-standards coding persists, with CMS systems and site-building services sometimes producing non-compliant code. An example is WIX, which transitioned from Flash to HTML5/JavaScript but often lacks essential structure and semantics.

What do Web Standards accomplish? Designing with Web Standards ensures that pages display as intended across various browsers, platforms, and devices, reducing the likelihood of crashes or limited rendering. Other benefits include lighter web pages, faster loading times, reduced bandwidth usage, and simplified development. Using external style sheets allows for diverse styles, responsive design, and easier site-wide changes. Additionally, Web Standards enhance search engine optimization, improve accessibility, and encourage the inclusion of alt text for images. External CSS facilitates centralized control over the presentation of multiple pages, making site-wide changes more efficient.
